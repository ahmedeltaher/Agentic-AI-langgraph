🏷️ مُزخرف الأدوات (@tool)

المدرب شرح فكرة الـ decorator:

🏷️ السحر في السطر الواحد

"The @tool decorator is used to set up this function as a tool for future use"
مُزخرف @tool بيتستخدم لتحضير الدالة دي كأداة للاستخدام المستقبلي!

وأكد على الفائدة الكبيرة من ده:

"This eliminates the step to explicitly create a function tool"
ده بيلغي الخطوة اللي كنا محتاجين فيها ننشئ أداة دالة صراحة!
ما هو الـ Decorator؟
الـ Decorator في Python هو طريقة لتعديل أو تحسين وظيفة دالة من غير ما نغير كودها. @tool بياخد دالة عادية ويحولها لأداة يقدر الوكيل يستخدمها!

🔄 قبل وبعد الـ Decorator:

❌ الطريقة القديمة:
إنشاء دالة، ثم إنشاء أداة منفصلة، ثم ربطهم ببعض
✅ الطريقة الجديدة:
إضافة @tool فوق الدالة وخلاص!
فائدة كبيرة:
الـ @tool decorator بيوفر علينا كتير من الكود المكرر والمعقد. بدل ما نكتب عشرات الأسطر لإنشاء أداة، نضيف سطر واحد فقط!

➕ الأداة الأولى: find_sum

المدرب بدأ بإنشاء أول أداة:

"The first tool that we will create is the find_sum tool"
أول أداة هننشئها هي أداة find_sum!
@tool def find_sum(x: int, y: int) -> int: """ This function is used to add two numbers and return their sum. It takes two integers as inputs and returns an integer as output. """ return x + y

🔍 تحليل الدالة:

🏷️ الـ Decorator: @tool يحول الدالة لأداة
📝 اسم الدالة: find_sum - واضح ومعبر
📥 المعاملات: x: int, y: int - رقمين صحيحين
📤 المخرج: -> int - رقم صحيح
📚 الـ Docstring: وصف مفصل للدالة
⚙️ المنطق: return x + y - عملية الجمع
لماذا Type Hints؟
الـ type hints (x: int, y: int) -> int مش مجرد تجميل! دي معلومات مهمة للوكيل عشان يعرف إيه نوع البيانات اللي يبعتها للدالة وإيه اللي يتوقعه في المقابل!

📚 أهمية الـ Docstring

المدرب أكد على أهمية الـ docstring:

🔑 العنصر الحاسم

"Inside the function, we use a docstring comment to describe the function. This docstring is critical for tool use"
جوه الدالة، نستخدم تعليق docstring لوصف الدالة. الـ docstring ده حاسم لاستخدام الأداة!

وشرح ليه ده مهم جداً:

"This description provided here is used by the LLM to determine the capabilities of this tool"
الوصف اللي متوفر هنا بيتستخدم من نموذج اللغة الكبير لتحديد قدرات الأداة دي!

وأدى نصيحة ذهبية:

"so be as elaborate and descriptive as possible when writing this docstring"
فكن مفصل ووصفي قدر الإمكان لما تكتب الـ docstring ده!
الـ LLM بيقرأ الـ Docstring!
الـ docstring مش للمبرمجين بس، ده كمان للـ LLM! نموذج اللغة بيقرأ الوصف ده عشان يفهم إمتى وإزاي يستخدم الأداة. كلما كان الوصف أوضح، كلما كان الوكيل أذكى في استخدام الأداة!

✖️ الأداة الثانية: find_product

المدرب انتقل للأداة الثانية بنفس النمط:

"In a similar fashion, we set up the second tool for find_product"
بنفس الطريقة، نجهز الأداة الثانية find_product!
@tool def find_product(x: int, y: int) -> int: """ This function is used to multiply two numbers and return their product. It takes two integers as inputs and returns an integer as output. """ return x * y

وأكد على وظيفتها:

"This returns the product of the two input integers, x and y"
دي بترجع حاصل ضرب الرقمين الصحيحين اللي دخلوا، x و y!

🔍 مقارنة بين الأداتين:

➕ find_sum
• العملية: الجمع (x + y)
• الاستخدام: "إضافة" أو "مجموع"
• المثال: 3 + 5 = 8
✖️ find_product
• العملية: الضرب (x * y)
• الاستخدام: "ضرب" أو "حاصل ضرب"
• المثال: 3 × 5 = 15
التشابه في التصميم:
لاحظ إن الدالتين لهم نفس البنية: نفس نوع المعاملات، نفس نوع المخرج، نفس نمط الـ docstring. ده مش صدفة! ده تصميم متسق يساعد الوكيل يفهم ويستخدم الأدوات بشكل أفضل!
ملاحظة على الـ Docstring:
لاحظ إن الـ docstring للأداة التانية مختلف ومناسب للعملية. مش نسخ ولصق، لا ده وصف دقيق لكل أداة على حدة!
