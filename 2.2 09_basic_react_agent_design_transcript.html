🔧 الأدوات المستخدمة

المدرب وضح إن الوكيل هيستخدم أدوات للعمليات الحسابية:

"To do the math, it uses tools"
عشان يعمل الحسابات، بيستخدم أدوات!

وحدد الأدوات المطلوبة:

➕ الأداة الأولى

find_sum()
أداة دالة للجمع

✖️ الأداة الثانية

find_product()
أداة دالة للضرب

📋 تفاصيل الأدوات:

➕ find_sum() - دالة الجمع

"Given two integers, this function returns their sum"
بإعطائها رقمين صحيحين، الدالة دي ترجع مجموعهما

function find_sum(a, b) { return a + b; }

✖️ find_product() - دالة الضرب

"This tool returns the product of two integers"
الأداة دي ترجع حاصل ضرب رقمين صحيحين

function find_product(a, b) { return a * b; }
فلسفة الأدوات:
الوكيل مش بيعمل الحسابات بنفسه، لا ده بيستخدم أدوات متخصصة! ده يخلي النظام قابل للتوسع - نقدر نضيف أدوات جديدة للقسمة أو الجذر أو أي عملية تانية!

💬 مثال على التفاعل

المدرب أدى مثال واضح على إزاي الوكيل هيشتغل:

📝 مثال عملي

"The user will provide an input, like What is the sum of 2 and 3?"
المستخدم هيقدم مدخل، زي "إيه مجموع 2 و 3؟"

🔄 تدفق العمل:

👤 مدخل المستخدم
"What is the sum of 2 and 3?"
→
🧠 تحليل LLM
تحديد العملية والأرقام
→
🔧 اختيار الأداة
find_sum()
→
⚡ تنفيذ
find_sum(2, 3)
→
📤 النتيجة
"The answer is 5"

⚙️ خطوات عمل الوكيل

"The agent will use an LLM to decide on the tools to use, then execute the tools, and return the results"
الوكيل هيستخدم نموذج اللغة الكبير ليقرر الأدوات اللي يستخدمها، وبعدين ينفذ الأدوات، ويرجع النتائج!

1 تحليل المدخل: LLM يقرأ ويفهم السؤال
2 استخراج المعلومات: تحديد العملية (جمع) والأرقام (2، 3)
3 اختيار الأداة: تحديد find_sum() كالأداة المناسبة
4 التنفيذ: استدعاء find_sum(2, 3)
5 إرجاع النتيجة: تقديم الإجابة "5" للمستخدم
