🤔 إيه هو LangGraph؟

المدرب سأل السؤال الأساسي: "What is LangGraph?" وأجاب بوضوح:

📊 تعريف LangGraph

"It is a framework developed in LangChain"
ده إطار عمل (framework) اتطور جوه LangChain

طب إيه هو LangChain أصلاً؟

🔗 LangChain - الأساس القوي

"LangChain is a popular framework for Generative AI"
LangChain هو إطار عمل مشهور للذكاء الاصطناعي التوليدي

نصيحة المدرب المهمة:
"It's recommended to learn about LangChain if you are not already familiar with it"
يُنصح بتعلم LangChain لو مش مألوف معاه!

يعني إيه إن LangGraph "developed in LangChain"؟ ده معناه إن LangGraph مش أداة منفصلة، لا ده جزء من عائلة LangChain الكبيرة!

العلاقة بين LangChain و LangGraph:
• LangChain = الأساس العام للذكاء الاصطناعي التوليدي
• LangGraph = التخصص في الأنظمة متعددة الوكلاء والـ workflows
• زي ما Windows هو نظام التشغيل، و Excel برنامج متخصص جواه

🎯 الهدف الأساسي من LangGraph

المدرب وضح الغرض الأساسي من LangGraph:

🏗️ إنشاء تطبيقات متعددة الوكلاء مع حفظ الحالة

"LangGraph helps to create stateful multi-agent applications"
LangGraph بيساعد في إنشاء تطبيقات متعددة الوكلاء مع حفظ الحالة

يعني إيه "stateful multi-agent applications"؟

👥 Multi-Agent

أكتر من وكيل ذكي يشتغلوا مع بعض في نفس التطبيق

💾 Stateful

بيحتفظ بالحالة والبيانات عبر خطوات التنفيذ المختلفة

📱 Applications

تطبيقات كاملة وقابلة للاستخدام العملي

🔧 Framework

إطار عمل متكامل يوفر كل الأدوات اللازمة

مثال توضيحي:
تخيل شركة استشارات فيها فريق من الخبراء كل واحد متخصص في مجال. العميل يجي بمشكلة معقدة، والشركة توزع أجزاء المشكلة على الخبراء المختلفين، وكل واحد يشتغل على جزءه، وفي النهاية النتايج تتجمع في حل شامل. ده بالضبط اللي LangGraph بيعمله!

🔗 التكامل مع LangChain

لكونه جزء من LangChain، بيتكامل كويس مع قدرات LangChain التانية

المدرب أدى مثال عملي على التكامل ده:

مثال عملي على التكامل:

مثلاً، أداة استرجاع (retriever) مبنية باستخدام LangChain ممكن تتكامل بسهولة كأداة في LangGraph!

ده معناه إيه؟ يعني لو انت مستخدم LangChain وعندك أدوات وأنظمة مبنية بيه، مش محتاج تبدأ من الصفر مع LangGraph!

🌟 المميزات الأساسية لـ LangGraph

المدرب سأل: "What are the key LangGraph features?" - إيه هي المميزات الأساسية لـ LangGraph؟

📊 تصميم سير العمل المبني على الرسوم البيانية

"It uses a graph based workflow design"
بيستخدم تصميم سير عمل مبني على الرسوم البيانية (graphs)

ليه الرسوم البيانية مهمة؟

قوة الرسوم البيانية:
"Graphs are used to represent the elements of the workflow and how they interact with each other"
الرسوم البيانية بتُستخدم لتمثيل عناصر سير العمل وإزاي بتتفاعل مع بعضها البعض!

تخيل سير العمل زي خريطة طرق، كل نقطة في الخريطة هي خطوة في العمل، والطرق هي العلاقات بين الخطوات!

نوع الرسم البياني:
"It is implemented as a directed graph"
بيتطبق كرسم بياني موجه (directed graph) - يعني الطرق لها اتجاه محدد!

🧩 مكونات الرسم البياني

المدرب وضح المكونات الأساسية للرسم البياني:

مكونات الـ Graph الأساسية:

⚙️ Processing Nodes (عقد المعالجة)
النقاط اللي بتحصل فيها المعالجة والعمل الفعلي
🔗 Edges (الحواف)
الروابط اللي بتوصل العقد ببعضها البعض
🤔 Conditional Edges (الحواف الشرطية)
روابط ذكية تسمح بالانتقال بناءً على فحوصات شرطية
وعد المدرب:
"We will explore them later in the course"
هنستكشفهم لاحقاً في الكورس!

مثال بسيط على التدفق:

البداية
→
معالجة
→
قرار
↙️ ↘️
مسار أ
مسار ب
مثال توضيحي:
تخيل نظام موافقة قرض في البنك:
• Node 1: فحص البيانات الأساسية
• Node 2: تحليل التاريخ الائتماني
• Conditional Edge: لو النتيجة جيدة → موافقة، لو سيئة → رفض
• Node 3أ: إجراءات الموافقة
• Node 3ب: إجراءات الرفض

لما prompt يوصل للـ agent، الـ graph بيتنفذ عبر العقد بتاعته

"create outputs"
بتنشئ مخرجات

وبعدين بتنقل التحكم للعقدة اللي بعدها

🔁 التكرار

العملية تتكرر لحد الوصول للنتيجة النهائية

مثال عملي:
مستخدم سأل: "عايز أحجز رحلة طيران لباريس"
• Node 1: فهم الطلب واستخراج المعلومات (الوجهة: باريس)
• Node 2: البحث في أسعار الطيران
• Node 3: عرض النتائج على المستخدم
كل عقدة تاخد نتيجة اللي قبلها وتبني عليها!

LangGraph بيساعد في إدارة الحالة أثناء تنفيذ سير العمل ويتتبع كل الخطط والمدخلات والمخرجات!

ده تصميم ذكي جداً! بدل ما كل agent يشتغل لوحده، كلهم بيبقوا جزء من نفس الـ workflow!
